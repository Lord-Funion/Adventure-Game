Adventure Game Code Guide

This file explains how to run the game, where to make common edits, what every active Python file does, and what the important functions and variables control.

Run The Game

The active version is the reorganized Python game. Run it from this project folder:

python3 main.py

The Web-Port branch also includes an HTML5 terminal-style browser version. Run it locally:

python3 -m http.server 4173

Then open:

http://127.0.0.1:4173/

In the web version, numbered menu rows and combat spell rows can be clicked. Typing the number, command, or spell still works too.

To host it at lordfunion.dev/adventure-game, upload index.html and web/ from this branch into public_html/adventure-game/.

Load a local encrypted save directly:

python3 main.py saves/autosave.tasave

Speed up or skip the text pauses while testing:

TEXT_ADVENTURE_SPEED=fast python3 main.py
TEXT_ADVENTURE_SPEED=instant python3 main.py

Hide the startup logo while testing:

TEXT_ADVENTURE_HIDE_LOGO=1 python3 main.py

How To Edit The Code

Edit the active game files in text_adventure/. The older one-file game is preserved on the legacy git branch instead of in a folder on main.

Common Changes

Starting money, health, mana, armor, backpack, or spells text_adventure/player.py The dictionary returned by create_player().

Add A New Story Scene

  1. Create a new function in text_adventure/story.py, for example dragon_scene(player).
  2. Add the scene id to SCENE_ORDER, for example "dragon".
  3. Add a readable title to SCENE_TITLES.
  4. Add an elif branch in _run_scene() that calls your new scene function.
  5. Use helpers like yes_no(), fight_or_run(), choose_left_or_right(), spell_fight(), and offer_potions() instead of rewriting input loops.

Add A New Spell

  1. Add a new entry in text_adventure/data.py under SPELLS.
  2. Use "damage" for attack spells, "healing" for healing spells, and "manaCost" for the cost.
  3. Optional effects currently supported by combat are "burn" and "stun".
  4. Teach the spell in a scene with add_spell(player, "Spell Name"), or add a shop row in run_shop().

Add A New Monster

  1. Add a new entry in text_adventure/data.py under MONSTERS.
  2. Give it "health", "damage", and an "attacks" list.
  3. Start the fight from a scene with spell_fight("monster_name", player).

Add A New Shop Item

  1. If it is a one-time item, add it to _create_shop_stock() in story.py.
  2. Add a MenuOption for it inside run_shop() in shop.py.
  3. Add a matching elif branch after the menu choice to call _buy_item(), _buy_spell(), or _buy_equipment().

Game State Variables

Most functions pass around a player dictionary or a full state dictionary. These are the most important variables in the project.

state Dictionary

Key Meaning
state["player"] The full player dictionary. This holds money, health, mana, armor, inventory, and learned spells.
state["shop_stock"] A dictionary that tracks one-time shop purchases. True means still available. False means already bought or learned.
state["next_scene"] The scene id that should run next. It must be in SCENE_ORDER or equal FINISHED_SCENE.

player Dictionary

Key Starting value What it controls
money 0 Gold used to buy spells, potions, mana, armor, and boots. Fights and treasure increase it.
health 100 Current hit points. If this reaches 0 or lower, game_over() ends the game.
healthMax 100 Maximum health. Big potions restore health to this value, and small potions cannot heal above it.
mana 50 Current spell energy. Combat spells subtract their manaCost from this.
manaMax 50 Maximum mana. Buying mana raises both mana and manaMax. Winning a fight restores mana to this value.
armor 0 Damage reduction against normal monster attacks. Poison still applies its status damage.
extraDamage 0 Extra damage added to damaging spells. Mage Boots increase this by 3.
backpack [] Inventory list. It stores items such as the frog, wand, potions, loot, helmet, and boots.
spells [] List of learned spell names. Combat only offers spells from this list that exist in SPELLS.

Variable Where it appears What it does

shop_stock / stock story.py, shop.py Tracks whether one-time shop items and spells are still available.

Spells, Monsters, And Loot

SPELLS entries

Field What it means
damage Base damage dealt to the monster. extraDamage is added during combat. 0 damage is allowed for control spells.
healing Health restored to the player during combat. Healing cannot go above healthMax.
manaCost Mana required to cast the spell in combat.
effects Optional dictionary of extra effects. Current combat logic understands burn and stun.
effects["burn"] Number of turns the target keeps burning. Each turn deals STATUS_DAMAGE.
effects["stun"] Number of monster turns skipped.
description Short explanation used in shop menus and for human readability.

MONSTERS entries

Field What it means
health Monster hit points at the start of combat.
damage Normal damage before player armor reduces it.
attacks List of attack names. One is picked randomly each monster turn.

Loot Lists

Variable What it does
LOOT_DROPS Random list used by win_fight(). Winning a fight picks one item from this list and adds it to the backpack.
SELLABLE_LOOT Set of junk items that shops automatically sell for random money. Potions are not included, so they are kept.

Legacy Branch

The older one-file version is preserved on the legacy git branch instead of in a folder on main.

Switch to the legacy branch and run it from the project root:

git switch legacy
python3 main.py

Return to the current modular game:

git switch main

Save And Cloud Notes

The game creates encrypted checkpoint saves in saves/. Checkpoints appear between story scenes, and the main menu can load existing .tasave files.

Save files are encrypted and signed with a tamper check. Editing money, health, inventory, or spells directly inside a save file will make the save fail to load. To change those values for development, edit create_player(), scene rewards, shop prices, or combat rewards instead.

Cloud saves are optional. The game writes local .tasave files first, and local loading still works when the web API is offline. Use Cloud Saves from the main menu or checkpoint menu to create an account, sign in, upload the current checkpoint, or download a cloud save.

The default cloud API is https://lordfunion.dev/adventure-api. For testing another API URL, run:

ADVENTURE_API_URL=https://example.com/adventure-api python3 main.py
